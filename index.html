IxD

“Good [examples of digital interaction design] oscillate between hiding and revealing themselves.”

– David Bolter & Diane Gromala

Concept

To me, the above quote can be interpreted in two ways:

  • Good design is both present and absent at the same time. Some of the best designs are never noticed because they were intended to not be noticed, but instead contribute to the overall experience.
  • Good user interaction design is comprised of elements that are more subdued when the user doesn’t need them, but when the user does, they become apparent.

Prompted by the quote and my interpretation, I have chosen to create a responsive website, with a system of interactive elements that are present when the user needs them, and are subdued or altogether disappeared when the user does not need them. To me, this is very important due to the vast amounts of information and varying screen sizes that exist today.

In this site, I have implemented a navigation system that disappears / reappears when needed (disappears when scrolling through a long page, reappears when just starting to scroll back up). You can see this disappearing / reappearing navigation bar simply by scrolling through this page. It’s subtle, but it makes all the difference when you are browsing on a mobile device and have limited screen real estate. I have only seen this done with the Google Search bar in Google Now.

Additionally, I created icon/button hybrids with various hover states to indicate varying levels of importance or relevance to the user and his / her experience. They are icons when not activated and transition to fuller buttons when hovered over so that the user is not interrupted with shiny buttons when they are trying to read through longer pieces of text. The best example of this that I’ve found is in Google Plus, and I wanted to try my hand at it myself.

Icon / Button Hybrids

Here are some sample icons. When each is hovered over, it will transform into a button. Since this is a proof of concept, they don’t link to anything.

Note: since there is no way to hover when using a touch-based device, I have implemented icons that are (flat) icons by default, and (shiny, rounded) buttons when pressed.

Process

In order to achieve the disappearing / reappearing header, I used the following CSS: 

#header {
	background-color: #1199aa; /* #17b6c3 */
	position: fixed; 
	top: 0;
	left: 0;
	width: 100%;
}

html.touch #header {
	-webkit-transition-property: margin-top;
 	-webkit-transition-duration: 0.25s;
	transition-property: margin-top;
 	transition-duration: 0.25s;
 }

#navigation {
	height: 60px;
	-moz-box-shadow: 0px 0px 15px 0px rgba(0, 0, 0, 0.18);
	-webkit-box-shadow: 0px 0px 15px 0px rgba(0, 0, 0, 0.18);
	box-shadow: 0px 0px 15px 0px rgba(0, 0, 0, 0.18);
}

...as well as the following JavaScript: 

$(window).ready(function() { 
	var header = $("#header");
	var lastTop = 0;  // the last px at the top of the browser
	var headerTop = 0;  // the top of the header (depends on how much is visible)
	var headerBottom = headerTop + 60;  // the bottom of the header
	$(window).scroll(function() {
		var top = $(window).scrollTop();  // tells you where the page is
		if (top < 0) {
			top = 0; // compensates for bouncing
		}
		if (top < headerTop) {
			headerTop = top; 
			headerBottom = headerTop + 60;
		} else if (top > headerBottom) {
			headerBottom = top - 1;  // so it's just 1px above the top of the browser
			headerTop = headerBottom - 60;  // so it is hidden
		} 
		var marginTop = headerTop - top;
		header.css('margin-top', marginTop + 'px'); // format turns it into a CSS variable
		lastTop = top;  // set the last top to be the current top
	});	
});

For the icon/button hybrids, I used multiple div elements for each icon: 

<div class="icon favorite">
	<div class="backgroundWrapper">
		<div class="background">
		</div>
	</div>
	<img src="images/favorites.png" alt="star" height="30" width="30"/>
</div>

In order to achieve the fade effect, used the following CSS: 

				
.icon {
	height: 45px;
	width: 45px;
	margin: 0 15px 15px 0;	
	position: relative;
	cursor: pointer;
	float: left;
}

.background {
	height: 43px;
	width: 43px;
	-moz-border-radius: 9px;
	border-radius: 9px;
	background: -webkit-gradient(linear, left top, left bottom, from(#e5e5e5), to(#d2d2d2));
}

.backgroundWrapper {
	position: absolute;
	left:0;
	top:0;
	height: 43px;  /* 1px padding makes it 45px */
	width: 43px;   /* 1px padding makes it 45px */
	padding: 1px;
	background: -webkit-gradient(linear, left top, left bottom, from(#d5d5d5), to(#c2c2c2));
	-moz-border-radius: 10px;
	border-radius: 10px;
	z-index: -1;
	-webkit-transition-property: opacity;
 	-webkit-transition-duration: 1s;
	transition-property: opacity;
 	transition-duration: .5s;
 	opacity: 0;
}

.icon:hover .backgroundWrapper, .icon.hover_effect .backgroundWrapper {
	opacity: 1;
}

.icon img {
	display: block;
	margin: 0 auto 0 auto;
	padding-top: 7px;
}

And to make it work (kind of) with a touch-based device, I used the following JavaScript: 

	
$('.icon').bind('touchstart touchend', function(e) {
	e.preventDefault();
	$(this).toggleClass('hover_effect');
});
